Improving the experience of the Glover mobile application

Overview

I redesigned the Glover mobile experience to reduce KYC-related drop-offs, introduce alternative payment methods, and streamline onboarding. By making KYC optional for specific services and improving navigation, we increased user retention by 10% and improved app ratings from 3.5 to 4.2 stars.

Glover is a multifunctional financial utility app that allows users to:

  1. Purchase gift cards
  2. Convert airtime to cash
  3. Buy data & internet subscriptions
  4. Pay electricity bills
  5. Purchase e-tickets
  6. Use Perfect Money and other payment options

The app aims to serve as a one-stop hub for digital transactions.

Role

Product Designer (UX/UI)

Responsible for research, mapping user pain points, redesigning user activation flows, and improving the app structure to increase retention.

Client

Glover

Year

2020-present

Category

Mobile Banking / Fintech

Problem / Challenge

Key Issue: High Drop-Off During KYC (BVN Entry)

We observed a major activation problem:

30% of users abandoned the app during the mandatory KYC step, citing:

  1. Lack of trust in entering BVN
  2. Privacy and security concerns
  3. Unclear reasons for verification
  4. The requirement blocking access to most features
Additional Retention Challenges
  1. No alternative payment methods
  2. Confusing navigation
  3. Limited access to basic services prior to KYC
  4. Users unable to continue transactions after leaving the app
  5. Difficulty retrieving receipts when re-opening

These issues collectively reduced activation, engagement, and long-term retention.

Approach / Process

1. Research & Analysis

We conducted:

  1. User interviews
  2. In-app survey
  3. Competitive benchmarking (vs giftcards, egifter)
Key Insights
  1. Users wanted choice in how they pay
  2. Many were uncomfortable providing BVN upfront
  3. Competitors allowed access to basic transactions without full verification
  4. Users requested clearer instructions and receipt access

2. Hypothesis

If we remove mandatory KYC for non-sensitive services and introduce more flexible payment methods, then:

  1. More users will complete activation
  2. User retention will increase
  3. App trust will improve
  4. Engagement across transaction services will grow

This became our core design direction.

3. Design Process

1Site Map Redesign – Created a cleaner, frictionless site map:

  1. Reduced steps to access services
  2. Grouped core utilities (airtime, data, electricity, etc.)
  3. Moved KYC into a “progressive onboarding” flow

2Wireframes and Prototypes – Designed low-fidelity wireframes for rapid validation. Built interactive prototypes outlining:

  1. Optional KYC path
  2. New payment flow
  3. Receipt retrieval workflow

Conducted usability testing to validate ease of:

  1. Navigation
  2. Payment selection
  3. Account verification flow

3Feature Implementation

  1. Introduced alternative payment methods
  2. Made KYC optional for basic services
  3. Improved receipt history and retrieval access
  4. Added clearer microcopy explaining why KYC is required for certain features

4. Solution

1Optional KYC Flow – Instead of forcing users through BVN entry, we:

  1. Allowed use of basic services without it
  2. Used progressive disclosure to introduce KYC only when required
  3. Added privacy explanations to build trust

2Alternative Payment Methods – Added:

  1. Card payments
  2. Wallet balance

    3. This solved a major retention barrier.

3Improved Navigation & App Structure

  1. Simplified the home layout
  2. Reduced friction by removing unnecessary steps
  3. Created a clearer path to all major services

Outcomes / Results

  • App Store Rating Improved from 3.5 → 4.2 stars - Driven by reduced frustration and clearer flows.
  • User Retention Increased by 10% - The combination of optional KYC + alternative payment methods kept users active.
  • Drop-off during KYC significantly reduced - Users could now explore and transact before fully verifying.
  • Clearer communication improved trust - Privacy-focused microcopy and transparent flow design addressed user concerns.

5. Challenges and Learnings

Balancing Security With Convenience

KYC is necessary—but forcing it early created drop-offs.

We learned that timely, contextual, progressive onboarding is key.

Building Trust With Transparency

Explaining why BVN is requested reduced user anxiety.

Frictionless Access > Full Verification

Fintech users respond better when they can try the product before deep verification.

Conclusion

These improvements position Glover as a more user-friendly, competitive fintech product focused on accessibility and trust.
